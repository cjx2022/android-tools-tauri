================================================================================
                                    功能介绍
================================================================================

本工具集提供以下功能模块：

1. 设备管理
   - 自动检测 ADB 路径（优先使用程序目录下的 adb.exe 或 platform-tools 文件夹）
   - 手动设置 ADB 路径
   - 网络连接安卓设备（支持指定端口）
   - 批量导入 IP 地址连接设备
   - 断开设备连接
   - 设备列表显示（设备ID、状态、型号）

2. APK 安装
   - 支持批量安装 APK 到多个设备
   - 安装进度显示
   - 安装日志记录

3. 显示控制
   - 隐藏/显示状态栏
   - 隐藏/显示导航栏
   - 一键隐藏/显示所有栏

4. 文件管理
   - 上传文件到安卓设备
   - 从安卓设备下载文件
   - 安卓文件浏览器

5. 批量删除 (v1.0.0 新增功能)
   - 在设备列表中选择要操作的设备
   - 输入或浏览选择要删除的文件或文件夹路径
   - 一键批量删除多个设备上的指定路径
   - 删除日志实时显示
   - 删除失败或文件不存在的设备列表
   - 支持复制失败设备列表

================================================================================
                                    ADB 配置说明
================================================================================

本程序需要 ADB (Android Debug Bridge) 工具才能正常工作。

【自动检测方式】

程序会按以下顺序自动查找 adb.exe：

1. 程序所在目录下的 adb.exe
2. 程序所在目录下的 platform-tools\adb.exe
3. 系统 PATH 环境变量中的 adb.exe

【手动设置方式】

如果自动检测失败，程序会提示"未找到ADB，请手动设置"，此时可以：
- 点击"手动设置ADB"按钮，选择 adb.exe 文件的位置

【获取 ADB】

adb.exe 包含在 Android SDK Platform Tools 中，您可以从以下地址下载：
https://developer.android.com/studio/releases/platform-tools

或者从 Android Studio 的安装目录中找到：
<Android SDK 路径>\platform-tools\adb.exe

【推荐配置】

将 platform-tools 整个文件夹复制到程序所在目录，例如：

发布版本v1.0.2/
├── android-tools.exe
├── platform-tools/
│   ├── adb.exe
│   ├── fastboot.exe
│   └── ...
└── readme.txt

这样程序会自动检测到 platform-tools\adb.exe。

【验证 ADB 是否可用】

1. 打开命令提示符 (CMD)
2. 输入: adb version
3. 如果显示版本信息，说明 ADB 已正确安装

================================================================================
                                    使用说明
================================================================================

【批量删除功能使用步骤】

1. 确保 ADB 路径已正确设置（程序会自动检测同目录或 platform-tools 文件夹下的 adb.exe）

2. 在"设备列表"中勾选需要删除文件的设备
   - 可以勾选多个设备进行批量操作
   - 只有在线设备才能执行删除操作

3. 在"批量删除"模块中：
   - 在"删除路径"输入框中输入要删除的文件或文件夹路径
   - 或者点击"浏览..."按钮，在安卓设备上浏览选择要删除的文件或文件夹
   - 支持删除单个文件或整个文件夹（递归删除）

4. 点击"批量删除"按钮
   - 系统会弹出确认对话框，请仔细确认删除路径
   - 确认后将对所有选中的设备执行删除操作

5. 查看删除结果：
   - "删除日志"区域会显示每个设备的删除结果
   - 如果有删除失败或文件不存在的设备，会在下方"删除失败或文件不存在的设备"区域列出
   - 可以点击"复制列表"按钮复制失败设备信息

【注意事项】

1. 删除操作不可恢复，请谨慎操作！
2. 删除前请确认路径正确，避免误删重要文件
3. 系统路径（如 /system/ 下的文件）可能需要 root 权限才能删除
4. 建议先测试删除一个设备，确认无误后再批量操作
