================================================================================
                                    功能介绍
================================================================================

本工具集提供以下功能模块：

1. 设备管理
   - 自动检测 ADB 路径（优先使用程序目录下的 adb.exe 或 platform-tools 文件夹）
   - 手动设置 ADB 路径
   - 网络连接安卓设备（支持指定端口）
   - 批量导入 IP 地址连接设备
   - 断开设备连接
   - 设备列表显示（设备ID、状态、型号）
   - 批量重启设备

2. APK 批量安装和卸载 (v1.4.0 新增批量卸载)
   - 支持批量安装 APK 到多个设备
   - 支持批量卸载应用（默认包名：com.arcsoft.arcfacedemo）
   - 安装/卸载进度显示
   - 操作日志记录

3. 显示控制（仅适用于安卓10及以下）
   - 隐藏/显示状态栏
   - 隐藏/显示导航栏
   - 一键隐藏/显示所有栏

4. 文件管理
   - 上传文件到安卓设备
   - 从安卓设备下载文件
   - 安卓文件浏览器

5. 批量删除
   - 在设备列表中选择要操作的设备
   - 输入或浏览选择要删除的文件或文件夹路径
   - 一键批量删除多个设备上的指定路径
   - 删除日志实时显示
   - 删除失败或文件不存在的设备列表
   - 支持复制失败设备列表

6. APP 配置备份与批量恢复 (v1.3.0 新增)
   - 获取前台应用包名
   - 备份应用配置数据到本地
   - 批量恢复配置到多个设备
   - 自动修复权限

7. 屏幕镜像 (v1.2.0 新增)
   - 远程查看安卓设备屏幕
   - 集成 scrcpy 工具

8. ADB 命令行 (v1.4.1 新增)
   - 选择一台设备执行 ADB 命令
   - 支持输入任意 ADB 命令
   - 终端风格输出显示
   - 实时查看命令执行结果

9. 界面功能
   - 浅色/深色主题切换
   - 模块折叠/展开功能
   - 折叠状态自动记忆

================================================================================
                                    ADB 配置说明
================================================================================

本程序需要 ADB (Android Debug Bridge) 工具才能正常工作。

【自动检测方式】

程序会按以下顺序自动查找 adb.exe：

1. 程序所在目录下的 adb.exe
2. 程序所在目录下的 platform-tools\adb.exe
3. 程序所在目录下所有子文件夹中的 adb.exe
4. 系统 PATH 环境变量中的 adb.exe

【手动设置方式】

如果自动检测失败，程序会提示"未找到ADB，请手动设置"，此时可以：
- 点击"手动设置ADB"按钮，选择 adb.exe 文件的位置

【获取 ADB】

adb.exe 包含在 Android SDK Platform Tools 中，您可以从以下地址下载：
https://developer.android.com/studio/releases/platform-tools

或者从 Android Studio 的安装目录中找到：
<Android SDK 路径>\platform-tools\adb.exe

【推荐配置】

将 platform-tools 整个文件夹复制到程序所在目录，例如：

发布版本v1.4.1/
├── 安卓工具集.exe
├── platform-tools/
│   ├── adb.exe
│   ├── fastboot.exe
│   └── ...
├── scrcpy-win64-v3.2/    (屏幕镜像功能需要)
│   ├── scrcpy.exe
│   └── ...
└── readme.txt

这样程序会自动检测到 platform-tools\adb.exe。

【验证 ADB 是否可用】

1. 打开命令提示符 (CMD)
2. 输入: adb version
3. 如果显示版本信息，说明 ADB 已正确安装

================================================================================
                                    使用说明
================================================================================

【基本使用流程】

1. 确保 ADB 路径已正确设置（程序会自动检测）
2. 连接设备（USB连接或网络连接）
3. 在设备列表中选择要操作的设备
4. 使用各功能模块进行操作

【设备连接方式】

方式一：USB 连接
- 使用 USB 线连接安卓设备到电脑
- 在安卓设备上开启 USB 调试模式
- 程序会自动识别设备

方式二：网络连接
- 在"ADB连接设置"模块中输入设备 IP 地址和端口（默认5555）
- 点击"连接"按钮
- 支持批量导入 IP 地址

【模块折叠功能】

- 点击模块标题栏可以折叠/展开模块
- 使用左上角的"折叠全部"按钮一键折叠所有模块
- 折叠状态会自动保存，下次打开时恢复

【主题切换】

- 点击右上角的主题切换按钮可以在浅色/深色主题间切换
- 主题偏好会自动保存

【批量删除功能使用步骤】

1. 在"设备列表"中勾选需要删除文件的设备
   - 可以勾选多个设备进行批量操作
   - 只有在线设备才能执行删除操作

2. 在"批量删除"模块中：
   - 在"删除路径"输入框中输入要删除的文件或文件夹路径
   - 或者点击"浏览..."按钮，在安卓设备上浏览选择要删除的文件或文件夹
   - 支持删除单个文件或整个文件夹（递归删除）

3. 点击"批量删除"按钮
   - 系统会弹出确认对话框，请仔细确认删除路径
   - 确认后将对所有选中的设备执行删除操作

4. 查看删除结果：
   - "删除日志"区域会显示每个设备的删除结果
   - 如果有删除失败或文件不存在的设备，会在下方列出
   - 可以点击"复制列表"按钮复制失败设备信息

【APP配置备份与恢复使用步骤】

1. 配置备份：
   - 在设备列表中选择一台设备（只能选择一台）
   - 点击"显示前台应用包名"获取当前运行的应用
   - 点击"备份配置数据"，选择保存路径
   - 备份文件将保存为 .tar.gz 格式

2. 批量恢复：
   - 在设备列表中选择多台设备
   - 选择备份文件
   - 输入目标包名
   - 点击"批量恢复"按钮
   - 程序会自动上传、解压、修复权限

【屏幕镜像使用】

1. 确保 scrcpy 已放置在程序目录下
2. 在设备列表中选择一台设备
3. 点击"屏幕镜像"按钮
4. 会弹出窗口显示设备屏幕

【ADB 命令行使用】

1. 在设备列表中选择一台设备（只能选择一台）
2. 找到页面底部的"ADB 命令行"模块
3. 在命令输入框中输入 ADB 命令
4. 点击"执行"按钮或按回车执行
5. 查看下方输出区域的命令执行结果

示例命令：
- shell pm list packages    # 列出所有已安装包
- shell ls /sdcard/         # 查看 sdcard 目录内容
- shell ps                  # 查看进程列表
- pm install -r xxx.apk     # 安装 APK

【批量卸载使用步骤】

1. 在设备列表中勾选需要卸载应用的设备
2. 在"APK批量安装和卸载"模块中：
   - 输入要卸载的应用包名（默认为 com.arcsoft.arcfacedemo）
3. 点击"批量卸载"按钮
4. 确认提示后执行卸载
5. 查看卸载日志

================================================================================
                                    注意事项
================================================================================

1. 删除操作不可恢复，请谨慎操作！
2. 删除前请确认路径正确，避免误删重要文件
3. 系统路径（如 /system/ 下的文件）可能需要 root 权限才能删除
4. 建议先测试操作一个设备，确认无误后再批量操作
5. 显示控制功能仅适用于安卓10及以下版本
6. 部分功能需要设备有 root 权限才能正常使用
